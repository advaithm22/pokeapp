# Poker

No-Limit Texas Hold'em for up to 10 anonymous players, with a provably-fair shuffle.

## Stack

- **Server**: Node + TypeScript, Fastify (HTTP), Socket.IO (realtime)
- **Client**: React + Vite
- **State**: in-memory (play money — restart wipes balances)
- **Auth**: none. Each browser is issued a random 32-byte session token stored in `localStorage`. Pick a nickname per visit.

## Run it

```bash
cd app
npm install
npm run dev          # server on :3001, client on :5173
```

Open `http://localhost:5173` in two or more tabs (or browsers), pick a nickname in each, and a hand starts as soon as 2+ players are seated.

For production:

```bash
npm run build
npm start            # serves built client from server on :3001
```

## How the provably-fair shuffle works

Every hand follows a **commit / contribute / reveal** protocol:

1. **Commit.** When a hand begins, the server picks a 32-byte `serverSeed` and broadcasts `commit = SHA-256(serverSeed)` to all seated players (visible in the *Provably-fair shuffle* panel).
2. **Contribute.** Each seated client posts a 32-byte `clientSeed` (browser `crypto.getRandomValues`). Players who don't post in time get a deterministic fallback (still public).
3. **Combine.** The shuffle uses
   ```
   combinedSeed = SHA-256( serverSeed || ⨁ length-prefixed clientSeeds sorted by seat )
   ```
4. **Shuffle.** Fisher-Yates over a 52-card deck. The random index for swap step `i` is generated by a counter-based PRF — `HMAC-SHA-256(combinedSeed, "shuffle:i:attempt")` — with rejection sampling to avoid modulo bias.
5. **Reveal.** When the hand ends, the server reveals `serverSeed`. Anyone can:
   - confirm `SHA-256(serverSeed) == commit` (proves the server didn't change its mind), and
   - rerun the shuffle to confirm the deck order matches what was dealt.

Because the combined seed mixes in every player's contribution, **no single party — including the server — can predict the shuffle**, even if they cheat.

### Verifying a hand from the CLI

After a hand ends, open the *Provably-fair shuffle* panel. Copy the `serverSeed`, `commit`, and each seat's client seed, then:

```bash
npm run verify -- <serverSeedHex> <commit> 0:<clientSeed0> 1:<clientSeed1> ...
```

The script asserts the commit matches and prints the reproduced 52-card deck.

## Game rules

- No-Limit Texas Hold'em
- Blinds: $0.10 / $0.20 (all money is integer cents internally)
- Buy-in: each player picks a dollar amount on entry, min $4 (20 BB), max $1000
- 30-second action clock; auto-fold (or check) on timeout
- Standard side-pot handling for all-ins
- Heads-up: dealer is small blind; SB acts first preflop, BB first postflop
- **Add money to stack** any time. Mid-hand top-ups apply at the start of the next hand. All players see a notification in the log when someone tops up.
- **Away mode** any time. While away, you don't get dealt in. If at least one hand passes while you're away, you owe one BB on return — paid as a dead blind into the pot of your first hand back. (Simple approximation of "you missed a blind".)
- **Show cards** during the post-hand window: choose to show card 1, card 2, both, or muck. Useful for showing a bluff after winning uncontested.
- **Display unit toggle**: switch all amounts between `$X.XX` and `N BB` (persists per browser).

### 7-2 bonus

If a player **wins** a hand while holding any 7+2 hole cards (suited or off), they collect $1.00 from each non-winning opponent who was dealt into that hand. Wins by **showdown** or by **bluffing everyone out** both qualify. Each opponent's contribution is capped at their current stack.

## What's intentionally simple

- One global table (no lobbies).
- No persistence — server restart wipes everything. Play money only.
- No real-money / KYC / licensing concerns. Don't deploy this for gambling.
- Short all-ins reopen action under this simplified ruleset (strict rules say they shouldn't for previously-acted players). For play money this is unlikely to matter; fix the `act()` raise-handling in `server/src/table.ts` if you care.
